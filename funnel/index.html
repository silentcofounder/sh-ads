UTM Attribution Fix — Know Which Ad Led to Which Purchase

Current state: PostHog can filter by GCLID (confirming Google Ads origin) but can't distinguish Search from PMax, campaign from campaign, or keyword from keyword. All 451 purchases look identical. You're flying blind on attribution. Here's the full fix.
Step 1 — Verify Auto-Tagging is ON in Google Ads
You already have GCLID data in PostHog so auto-tagging is likely on, but verify:
Google Ads → Settings (gear icon) → Account Settings
→ Auto-tagging → tick "Tag the URL that people click through from my ad"
→ Save
Step 2 — Add Tracking Template to Google Ads (Account Level)
This appends UTM parameters to every ad click automatically across all campaigns:
Google Ads → Settings → Account Settings → Tracking
→ Tracking template field, enter:

{lpurl}?utm_source=google&utm_medium=cpc&utm_campaign={campaign}&utm_content={adgroup}&utm_term={keyword}&gclid={gclid}
This gives PostHog: source, medium, campaign name, ad group name, keyword, and GCLID on every single click.
Step 3 — Pass UTMs from Main Site → Booking Subdomain
UTMs drop when users navigate from shamirihealth.com to booking.shamirihealth.com. Add this JS to your main site landing pages:
// Paste in <head> of all landing pages on shamirihealth.com
document.addEventListener("DOMContentLoaded", function() {
  const p = new URLSearchParams(window.location.search);
  const keys = ['utm_source','utm_medium','utm_campaign',
                 'utm_content','utm_term','gclid'];
  const str = keys.filter(k=>p.has(k))
                   .map(k=>`${k}=${p.get(k)}`).join('&');
  if (!str) return;
  document.querySelectorAll('a[href*="booking.shamirihealth.com"]')
    .forEach(a => {
      a.href += (a.href.includes('?') ? '&' : '?') + str;
    });
});
Step 4 — Configure PostHog to Capture UTMs as Person Properties
PostHog captures UTMs automatically on $pageview events. Verify they're being stored as person properties for funnel filtering:
PostHog → Data Management → Properties
→ Search "utm_campaign" — should appear as a person property
→ If missing: check PostHog snippet includes autocapture: true

In your PostHog JS init:
posthog.init('YOUR_KEY', {
  api_host: 'https://app.posthog.com',
  autocapture: true,
  capture_pageview: true,  // captures UTMs on every pageview
  persistence: 'localStorage'
})
